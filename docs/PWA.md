# PWA Setup

Clawmander is a Progressive Web App (PWA) that can be installed on any device for a native-like experience.

## What's Included

- **Service worker** (`public/sw.js`) — generated by `next-pwa` at build time, enables offline caching
- **Web manifest** (`public/manifest.json`) — defines app name, icons, theme color, and display mode
- **Icons** — `icon-192.png` and `icon-512.png` generated from PIL
- **Meta tags** — in `pages/_document.js`: `theme-color`, `apple-mobile-web-app-capable`, manifest link

## Enabling PWA Install (requires HTTPS)

Browsers require HTTPS to register a service worker (except on `localhost`). For local network access over Tailscale, use an nginx reverse proxy with a Tailscale certificate.

### 1. Get a Tailscale Certificate

```bash
sudo mkdir -p /etc/ssl/tailscale
sudo tailscale cert \
  --cert-file /etc/ssl/tailscale/<your-hostname>.ts.net.crt \
  --key-file  /etc/ssl/tailscale/<your-hostname>.ts.net.key \
  <your-hostname>.ts.net
```

Find your hostname with:
```bash
tailscale status --json | python3 -c "import sys,json; print(json.load(sys.stdin)['CertDomains'][0])"
```

### 2. Install the nginx Config

```bash
sudo cp nginx-clawmander.conf /etc/nginx/sites-available/clawmander
sudo ln -sf /etc/nginx/sites-available/clawmander /etc/nginx/sites-enabled/clawmander
sudo nginx -t && sudo systemctl reload nginx
```

The nginx config (`nginx-clawmander.conf` in the repo root) proxies HTTPS on port 443 to the Next.js frontend on port 3000.

### 3. Access via HTTPS

Open `https://<your-hostname>.ts.net` in Chrome or Edge. The install icon will appear in the address bar.

On iOS Safari: tap the Share button → "Add to Home Screen".

### 4. Renewing the Certificate

Tailscale certificates expire. Renew with:
```bash
sudo tailscale cert \
  --cert-file /etc/ssl/tailscale/<your-hostname>.ts.net.crt \
  --key-file  /etc/ssl/tailscale/<your-hostname>.ts.net.key \
  <your-hostname>.ts.net
sudo systemctl reload nginx
```

## Regenerating Icons

Icons are generated using Python PIL. To regenerate:

```bash
cd frontend
python3 - <<'EOF'
from PIL import Image, ImageDraw, ImageFont
import os

sizes = [192, 512]
for size in sizes:
    img = Image.new('RGBA', (size, size), (0, 0, 0, 0))
    draw = ImageDraw.Draw(img)
    draw.rounded_rectangle([0, 0, size-1, size-1], radius=size//8, fill=(17, 24, 39, 255))
    font_size = int(size * 0.65)
    try:
        font = ImageFont.truetype('/usr/share/fonts/truetype/dejavu/DejaVuSans-Bold.ttf', font_size)
    except:
        font = ImageFont.load_default()
    bbox = draw.textbbox((0, 0), 'C', font=font)
    x = (size - (bbox[2]-bbox[0])) // 2 - bbox[0]
    y = (size - (bbox[3]-bbox[1])) // 2 - bbox[1]
    draw.text((x, y), 'C', font=font, fill=(16, 185, 129, 255))
    img.save(f'public/icon-{size}.png')
    print(f'Created icon-{size}.png')
EOF
```

## next.config.js

PWA is configured in `next.config.js` via `next-pwa`:

```js
const withPWA = require('next-pwa')({
  dest: 'public',      // outputs sw.js and workbox-*.js to public/
  register: true,      // auto-registers the service worker
  skipWaiting: true,   // activates new SW immediately on update
  disable: process.env.NODE_ENV === 'development',  // disabled in dev
});

module.exports = withPWA(nextConfig);
```

The service worker is **disabled in development** (`npm run dev`) to avoid caching issues. It is active in production (`npm run start`).
